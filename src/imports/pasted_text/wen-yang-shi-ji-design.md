“纹样市集”高保真详细设计说明
面向前端开发、交互实现与业务联调的交付版文档




文档定位  本说明用于指导“纹样市集”模块的一期开发实现，默认继承既有的视觉风格、左侧导航结构、交互语义与双语能力。

项目	说明
对应版本	锦绣智织 V1.0 / 市集交易一期
适用范围	左侧菜单新增“纹样市集”后的全部页面、弹层、状态与交易流转
已确定前置模块	智绘创作、设计提案、我的纹库
本期核心目标	打通“发布 - 浏览 - 申请授权 - 同意 - 支付 - 入库”的最小可交易闭环
默认授权模型	非独占 + 有期限 + 有范围 + 不可再转授权



1. 设计目标与范围
1.1 设计目标
在保持当前主界面风格不变的前提下，为左侧新增一级菜单“纹样市集”，形成清晰、稳定、可交易的纹样市场入口。
保证用户在 60 秒内理解“看什么、买什么、处理什么”，并在 4 步内完成关键业务动作。
将“纹样展示页”升级为“交易与授权闭环模块”，避免只展示不成交。
将授权时间、范围、支付时效、入库规则一次定义清楚，减少前后端与法务反复沟通。
1.2 本期设计范围
范围	是否纳入一期	说明	备注
市集首页	纳入	展示全部可交易纹样，并承接快速筛选与页签切换	一级入口
纹样详情页	纳入	展示与“我的纹库”一致的字段，并增加交易信息	全页详情
申请授权	纳入	买方向卖方发起授权申请	抽屉式表单
待我处理	纳入	卖方审批别人发来的授权申请	任务流视图
支付倒计时	纳入	卖方同意后，买方 2 小时内完成付款	超时自动取消
自动入库	纳入	支付成功后自动进入买方“我的纹库”	来源=授权获得
独家买断	不纳入	二期再评估	法务复杂度高
再转授权	不纳入	一期禁止	避免权属链条混乱

设计原则  全模块同时遵循 MECE 法则、STAR 原则、懒人原则，以及“上线十要件”中的水印、权限、日志、双语与安全要求。

2.2 “纹样市集”页内结构
采用“页面标题 + 任务快捷入口 + 页签 + 筛选栏 + 内容区”的单页结构。
不新增左侧二级菜单，避免层级增加过深。
通过页签承担“浏览”和“做任务”两类语义，符合主流产品模式。
页签	面向角色	核心用途	默认表现

待我处理	卖方	审批授权申请	任务列表



3. 一期交易规则定稿
3.1 统一交易原则
项目	说明
授权方式	一期仅支持非独占授权
授权期限	必须填写，不允许为空
授权范围	至少明确使用用途、产品品类、地域或渠道中的关键项
付款时效	卖方同意后，买方需在 2 小时内完成付款
超时处理	未付款自动取消，订单失效，授权名额释放
入库方式	支付成功后自动进入买方“我的纹库”
再发布	授权获得的纹样不得再次发布到市集
再授权	授权获得的纹样不得再次授权给第三方

3.2 默认授权模板
模板	适用场景	默认期限	关键约束
单项目授权	品牌提案、一次产品开发、单次空间项目	90 天内完成项目落地	对应 1 个项目，不得转给第三方使用
年度授权	常规文创商品、品牌联名、持续销售	1 年	限定产品品类与地域/渠道，可续费
限量授权	礼盒、节庆款、纪念版、限量衍生品	1 年	需同时填写数量上限，例如 100 / 500 / 1000 件

产品建议  一期不建议默认提供“永久授权”或“一个月授权”作为主模板。前者风险与定价复杂度高，后者通常不足以覆盖确认、打样、生产与销售周期。

5. 申请授权流程
5.1 买方主流程
1.查看详情
2.发起申请
3.等待卖方处理
4.卖方同意后 2 小时内完成付款
5.支付成功自动入库
5.2 “申请授权”抽屉结构

使用主体	使用主体名称	是	企业 / 机构 / 个人主体名称
使用用途	用途说明	是	用于卖方判断是否授权
产品信息	产品品类、销售渠道、地域范围	是	至少限制一个维度
改编选项	是否允许改编	是	布尔值

交互规则  “提交申请”按钮在必填项未完成前保持禁用；校验失败时自动聚焦到第一个错误字段。

7. 支付与入库规则
7.1 支付倒计时
卖方同意后，买方进入“待付款”状态，展示剩余时间 02:00:00 倒计时。
买方可执行“立即付款”或“取消订单”；超时自动失效。
超时后状态变更为“已失效”，并记录原因“超时未付款”。
7.2 自动入库
入库字段	取值	是否可编辑	说明
来源	授权获得	否	区别于智绘创作 / 提案锁定 / 本地上传
资产归属	授权资产	否	非版权所有资产
原所有者	卖方名称	否	保留原始权利归属
授权单号	系统生成	否	用于溯源
授权起止时间	按订单模板带入	否	到期后状态可变更
是否允许再发布	否	否	前端直接隐藏发布按钮
是否允许再授权	否	否	前端直接隐藏授权相关按钮

关键限制  授权获得的纹样可以在“设计提案”中继续使用，但不得再次确权、再次上架或再次授权。

11. 埋点、权限与异常态
11.1 埋点最小集
进入市集首页、切换页签、点击筛选、点击卡片、查看详情
发起授权申请、提交申请、卖方同意、卖方拒绝、买方点击付款、支付成功、支付超时
自动入库、下架纹样、编辑发布、查看证书、导出授权说明
11.2 权限与异常态
项目	说明
无权限菜单	直接隐藏，不展示空壳入口
授权获得资产	隐藏“再次发布 / 再次授权 / 再次确权”按钮
价格变化	下单前若价格变更，强制用户重新确认
纹样已下架	申请入口禁用，并提示“该纹样已下架”
订单过期	展示“订单已失效，请重新申请”
双语要求	所有状态、按钮、字段、提示语统一进入 i18n 词条

12. 交付结论
一句话定义  “纹样市集”不是单纯图片展示页，而是‘纹样展示 + 授权申请 + 审批流转 + 支付成交 + 自动入库’的一体化交易模块。

前端可据此直接开始拆页面、拆组件、拆状态与接口字段。
后端需优先配合实现订单状态机、倒计时、自动取消与自动入库。
法务与业务侧如需补充合同模板，可在不改 UI 主结构的前提下追加。
